# Challenges in Representation Learning: Facial Expression Recognition Challenge

## 1. კონკურსის მიმოხილვა და ამოცანა 
kaggle contest: **Facial Expression Recognition (FER2013)**.
ამოცანა მდგომარეობს იმაში, რომ მოცემული ადამიანის სახის სურათის მიხედვით მოდელმა უნდა შეძლოს 7 განსხვავებული ემოციის კლასიფიკაცია.

* **0**: Anger (ბრაზი)
* **1**: Disgust (ზიზღი)
* **2**: Fear (შიში)
* **3**: Happy (სიხარული)
* **4**: Sad (სევდა)
* **5**: Surprise (გაკვირვება)
* **6**: Neutral (ნეიტრალური)

ეს არის კლასიკური **Multi-class Image Classification** პრობლემა, სადაც ძირითადი გამოწვევა სურათების დაბალი გაფართოება (48x48) და კლასების მკვეთრი დისბალანსია.

## 2. მონაცემების ანალიზი და მომზადება (EDA & Dataset Phase)

### 2.1. მონაცემების სტრუქტურა
კაგლიდან ჩამოტვირთული ფაილები (`train.csv` და `test.csv`) არ შეიცავს ცალკეულ სურათებს ფოლდერებში. მონაცემები მოცემულია ტექსტურ ფორმატში. თითოეული ჩანაწერი შედგება:
* `emotion`: კლასის ციფრული იდენტიფიკატორი (0-6).
* `pixels`: სფეისებით გამოყოფილი 2304 ციფრი (რომელიც წარმოადგენს 48x48 სურათის პიქსელებს).

### 2.2. Class Imbalance
ჩატარებული პირველადი ანალიზის (EDA) შედეგად გამოვლინდა სერიოზული დისბალანსი კლასებს შორის.

ეს დისბალანსი მნიშვნელოვანი ფაქტორი იქნება მოდელის შეფასებისას, რადგან მხოლოდ Accuracy-ზე ორიენტირება შეიძლება საკმარისი არ აღმოჩნდეს.

### 2.3. PyTorch Custom Dataset
მონაცემების ეფექტური დამუშავებისთვის შევქმენი კლასი `FERDataset`, რომელიც ახდენს:
1.  ტექსტური პიქსელების `numpy` მასივად გარდაქმნას.
2.  მის გადაყვანას 48x48 ფორმატში.
3.  PIL Image-ად კონვერტაციას, რათა სამომავლოდ მარტივად გამოიყენოს `torchvision.transforms`-მა.

შევქმენი მეთოდი "get_dataloaders":
1. train-ის გასაყოფად train და validation დატასეტებად.
2. კლასების მკვეთრი დისბალანსის (Class Imbalance) გამო, გაყოფისას გამოყენებულ იქნა stratify=full_df['emotion'] (scikit-learn-იდან). ეს უზრუნველყოფს, რომ როგორც Train, ისე Validation კომპლექტებში ემოციების პროპორციული განაწილება იდენტური დარჩეს.
3. ფუნქცია აბრუნებს PyTorch-ის DataLoader ობიექტებს batch_size=64 პარამეტრით. სატრენინგო ნაკრებზე ჩართულია shuffle=True, რათა ყოველ ეპოქაზე მონაცემების რიგითობა აირიოს (მოდელმა თანმიმდევრობა რომ არ დაიზესტოს).

### 2.4. მონაცემების წინასწარი დამუშავება (Data Preprocessing)
საწყის ეტაპზე გამოვიყენე მინიმალისტური, სუფთა პრეპროცესინგი:
1. transforms.ToTensor(): გადაჰყავს PIL Image პიქსელების მნიშვნელობები [0, 255] შუალედიდან PyTorch ტენზორში, [0.0, 1.0] დიაპაზონში.
2. transforms.Normalize(mean=[0.5], std=[0.5]): ახდენს მონაცემების ცენტრირებას და მასშტაბირებას [-1.0, 1.0] დიაპაზონში. ეს ნაბიჯი კრიტიკულია ნეირონული ქსელისთვის, რადგან ის თავიდან გვარიდებს გრადიენტების აფეთქებას (Exploding Gradients) და უზრუნველყოფს ოპტიმიზატორის (Adam) ბევრად უფრო სწრაფ და სტაბილურ კონვერგენციას.

## 3. ექსპერიმენტები და მოდელების არქიტექტურები
გადავწყვიტე მივყვე იტერაციულ პრინციპს: მარტივიდან რთულისკენ. თითოეული ექსპერიმენტი ილოგება დამოუკიდებელი Run-ის სახით Weights & Biases (Wandb) პლატფორმაზე.

### 3.1. ექსპერიმენტების გეგმა
* **Run 1: MLP_Shallow_Baseline** (1 hidden layer 128 ნეირონი)
* **Run 2: MLP_Deep_Iter1** (არქიტექტურის გაღრმავება და ლეიერების იტერაციული დამატება)

---

### 3.2. Run 1: MLP_Shallow_Baseline (საწყისი წერტილი)

**მიღებული გადაწყვეტილება:** კვლევა დავიწყე უმარტივესი სტრუქტურით. პირველ მოდელად ავირჩიე ხაზოვანი Multi-Layer Perceptron (MLP) 1 hidden layer-ით (128 ნეირონი) და ReLU აქტივაციით. ვინაიდან MLP სურათს შლის 1D ვექტორად (2304), ეს მოდელი ავიღე საუკეთესო Baseline-ად, რათა შევაფასოთ ხაზოვანი შრეების ლიმიტები სურათებთან მუშაობისას.

**ჰიპერპარამეტრები:**
* Learning Rate: 0.001 (Adam)
* Batch Size: 64
* Epochs: 10

**შედეგები (Wandb Summary):**
* **Final Train Accuracy:** 57.09%
* **Final Train Loss:** 1.1638
* **Final Validation Accuracy:** 41.74%
* **Final Validation Loss:** 1.6585

**Wandb-ის Run History ანალიზი:**
მკაფიოდ გამოიკვეთა **Overfitting**-ის ეფექტი. მე-4 ეპოქამდე სავალიდაციო Loss მცირდებოდა და მიაღწია მინიმუმს (1.5528), თუმცა მე-5 ეპოქიდან `val_loss`-მა დაიწყო ზრდა და მე-10 ეპოქაზე 1.6585-ს მიაღწია, მაშინ როცა `train_loss` სტაბილურად ეცემოდა. მოდელმა დაიწყო სატრენინგო პიქსელების "ზეპირად დამახსოვრება" და დაკარგა გენერალიზაციის უნარი.

### 3.3. Run 2: MLP_Deep_Iter1 (არქიტექტურის გაღრმავება)

**მიღებული გადაწყვეტილება:** პირველ ექსპერიმენტში მიღებული Underfitting/Overfitting-ის გამო, გადავწვიტე მოდელის Capacity-ის გაზრდა. `models.py`-ში დავამატე მეორე hidden layer. ახალი სტრუქტურა განისაზღვრა შემდეგნაირად: Input (2304) -> FC1 (512) -> ReLU -> FC2 (256) -> ReLU -> Output (7). მიზანი იყო შემემოწმებინა, დაეხმარებოდა თუ არა ქსელს გაღრმავება უფრო აბსტრაქტული feature-ების სწავლაში.

**ჰიპერპარამეტრები:**
* Learning Rate: 0.001 (Adam)
* Batch Size: 64
* Epochs: 10

**შედეგები (Wandb Summary):**
* **Final Train Accuracy:** 67.04%
* **Final Train Loss:** 0.8724
* **Final Validation Accuracy:** 41.48%
* **Final Validation Loss:** 1.8121

**Wandb-ის Run History ანალიზი და შედარება Baseline-თან:**
* დამატებულმა შრემ მკვეთრად გააუმჯობესა მოდელის სატრენინგო სიზუსტე (ავიდა 67.04%-მდე), რაც ამტკიცებს, რომ უფრო ღრმა არქიტექტურას მეტი ინფორმაციის დამახსოვრება შეუძლია.
* თუმცა, სავალიდაციო კომპლექტზე სიზუსტე არ გაზრდილა (41.48%), ხოლო `val_loss` მნიშვნელოვნად გაუარესდა (1.6585 -> 1.8121). გრაფიკი აჩვენებს, რომ მე-5 ეპოქის შემდეგ მოდელი კატასტროფულად დაცილდა რეალობას და დაიწყო ხმაურის (Noise) დასწავლა. ხაზოვანი შრეების კიდევ უფრო გაზრდა გენერალიზაციას ვეღარ აუმჯობესებს.

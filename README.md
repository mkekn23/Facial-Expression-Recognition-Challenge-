# Challenges in Representation Learning: Facial Expression Recognition Challenge

## 1. კონკურსის მიმოხილვა და ამოცანა 
kaggle contest: **Facial Expression Recognition (FER2013)**.
ამოცანა მდგომარეობს იმაში, რომ მოცემული ადამიანის სახის სურათის მიხედვით მოდელმა უნდა შეძლოს 7 განსხვავებული ემოციის კლასიფიკაცია.

* **0**: Anger (ბრაზი)
* **1**: Disgust (ზიზღი)
* **2**: Fear (შიში)
* **3**: Happy (სიხარული)
* **4**: Sad (სევდა)
* **5**: Surprise (გაკვირვება)
* **6**: Neutral (ნეიტრალური)

ეს არის კლასიკური **Multi-class Image Classification** პრობლემა, სადაც ძირითადი გამოწვევა სურათების დაბალი გაფართოება (48x48) და კლასების მკვეთრი დისბალანსია.

## 2. მონაცემების ანალიზი და მომზადება (EDA & Dataset Phase)

### 2.1. მონაცემების სტრუქტურა
კაგლიდან ჩამოტვირთული ფაილები (`train.csv` და `test.csv`) არ შეიცავს ცალკეულ სურათებს ფოლდერებში. მონაცემები მოცემულია ტექსტურ ფორმატში. თითოეული ჩანაწერი შედგება:
* `emotion`: კლასის ციფრული იდენტიფიკატორი (0-6).
* `pixels`: სფეისებით გამოყოფილი 2304 ციფრი (რომელიც წარმოადგენს 48x48 სურათის პიქსელებს).

### 2.2. Class Imbalance

### 2.3. PyTorch Custom Dataset
მონაცემების ეფექტური დამუშავებისთვის შევქმენი კლასი `FERDataset`, რომელიც ახდენს:
1.  ტექსტური პიქსელების `numpy` მასივად გარდაქმნას.
2.  მის გადაყვანას 48x48 ფორმატში.
3.  PIL Image-ად კონვერტაციას, რათა სამომავლოდ მარტივად გამოიყენოს `torchvision.transforms`-მა.

შევქმენი მეთოდი "get_dataloaders":
1. train-ის გასაყოფად train და validation დატასეტებად.
2. კლასების მკვეთრი დისბალანსის (Class Imbalance) გამო, გაყოფისას გამოყენებულ იქნა stratify=full_df['emotion'] (scikit-learn-იდან). ეს უზრუნველყოფს, რომ როგორც Train, ისე Validation კომპლექტებში ემოციების პროპორციული განაწილება იდენტური დარჩეს.
3. ფუნქცია აბრუნებს PyTorch-ის DataLoader ობიექტებს batch_size=64 პარამეტრით. სატრენინგო ნაკრებზე ჩართულია shuffle=True, რათა ყოველ ეპოქაზე მონაცემების რიგითობა აირიოს (მოდელმა თანმიმდევრობა რომ არ დაიზესტოს).

### 2.4. მონაცემების წინასწარი დამუშავება (Data Preprocessing)
საწყის ეტაპზე გამოვიყენე მინიმალისტური, სუფთა პრეპროცესინგი:
1. transforms.ToTensor(): გადაჰყავს PIL Image პიქსელების მნიშვნელობები [0, 255] შუალედიდან PyTorch ტენზორში, [0.0, 1.0] დიაპაზონში.
2. transforms.Normalize(mean=[0.5], std=[0.5]): ახდენს მონაცემების ცენტრირებას და მასშტაბირებას [-1.0, 1.0] დიაპაზონში. ეს ნაბიჯი კრიტიკულია ნეირონული ქსელისთვის, რადგან ის თავიდან გვარიდებს გრადიენტების აფეთქებას (Exploding Gradients) და უზრუნველყოფს ოპტიმიზატორის (Adam) ბევრად უფრო სწრაფ და სტაბილურ კონვერგენციას.
